ðŸ”° Built-In Variables In Awk

ðŸ’¨ NR :- use for current count ex. $ awk '{print NR,$0}' data.txt
ðŸ’¨ NF: NF command keeps a count of the number of fields with in the current input record. 
ðŸ’¨ FS: FS command contains the field separator character which is used to divide fields on the input line. The default is â€œwhite spaceâ€, meaning space and tab characters. FS can be reassigned to another character (typically in BEGIN) to change the field separator. 
ðŸ’¨ RS: RS command stores the current record separator character. Since, by default, an input line is the input record, the default record separator character is a newline. 
ðŸ’¨ OFS: OFS command stores the output field separator, which separates the fields when Awk prints them. The default is a blank space. Whenever print has several parameters separated with commas, it will print the value of OFS in between each parameter. 
ðŸ’¨ ORS: ORS command stores the output record separator, which separates the output lines when Awk prints them. The default is a newline character. print automatically outputs the contents of ORS at the end of whatever it is given to print.
